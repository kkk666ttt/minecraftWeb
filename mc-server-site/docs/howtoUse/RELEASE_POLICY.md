# 版本与发布策略

本文档定义本项目版本号、更新日志、发布节奏的统一规则。

## 1. 版本号规则

使用三段式：主版本.次版本.修订版本（示例：1.3.2）

- major：重大改版、结构变化、UI/交互大幅升级
- minor：新功能新增、页面模块增强
- patch：缺陷修复、样式修正、兼容性优化

## 2. 何时更新 data/changelog.json

以下情况必须写入更新日志：

- major 变更
- minor 新功能
- 对用户可感知的重要修复

纯内部小改（文档错字等）可不写入。

## 3. 发布责任建议

- 开发者：完成功能与自测
- 维护者：执行发布检查清单
- 发布者：更新版本号与更新日志

可由同一人兼任，但流程不可省略。

## 4. 建议发布流程

1. 完成代码与数据修改
2. 执行 docs/RELEASE_CHECKLIST.md
3. 更新 data/changelog.json
4. 更新必要的资源版本参数
5. 启动本地服务复测
6. 提交发布

## 5. 回滚建议

若发布后出现严重问题：

1. 优先回退最近一次变更文件
2. 保留数据文件快照
3. 在更新日志标注回滚说明

## 6. 文档同步要求

当下列内容变化时，必须同步文档：

- 数据字段变化 -> 更新 docs/DATA_SCHEMA.md
- 发布流程变化 -> 更新 docs/RELEASE_CHECKLIST.md / docs/RELEASE_POLICY.md
- 部署方式变化 -> 更新 docs/DEPLOYMENT_GUIDE.md
