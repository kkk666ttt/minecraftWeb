# 故障排查手册

本文档覆盖本项目最常见故障：白屏、404、缓存不生效、数据加载失败。

## 1. 页面白屏

### 现象

- 页面打开后只有空白
- 或只有背景，没有内容

### 排查步骤

1. 打开浏览器开发者工具，查看 Console 第一条红色报错
2. 打开 Network，检查是否存在 js/modules 或 data 下资源 404
3. 确认在正确目录启动（含 index.html）
4. 强制刷新：Ctrl+F5

### 高概率原因

- JS 语法错误（例如某模块报错导致 init 中断）
- JSON 格式错误（逗号、引号、数组结构问题）
- 某个 data/*.json 返回 404

## 2. 出现 304

### 结论

304 不是错误，是浏览器缓存命中。

### 处理方式

- 正常可忽略
- 如果你刚改了代码但没生效：
  1. Ctrl+F5 硬刷新
  2. 更新 index.html 里对应资源版本号

## 3. 出现 404

### 判断影响

- favicon.ico 404：通常不影响功能
- js/css/json/image 404：会导致功能异常或白屏

### 快速定位

看 Network 中 404 的完整 URL，再对照项目目录检查实际文件是否存在。

## 4. 本机能开，其他电脑打不开

### 常见原因

- 服务绑定了 127.0.0.1
- 使用了 localhost 而不是服务器内网 IP
- 防火墙未放行端口

### 正确启动

```bash
python3 -m http.server 8080 --bind 0.0.0.0
```

客户端访问：<http://服务器内网IP:8080>

## 5. Python 启动失败

### 检查

```bash
python3 --version
```

若不存在，请安装 python3。

## 6. 端口占用

### 检查

```bash
ss -lntp | grep :8080
```

### 处理

- 换端口（例如 8090）
- 或停止占用进程

## 7. JSON 修改后页面异常

### 常见错误

- 最后一项后多逗号
- 引号不成对
- 字段拼写错误
- 类型不匹配（如 tags 写成字符串）

### 建议

修改后先检查 JSON 格式，再刷新页面。

## 8. 画廊标签不显示

当前逻辑：标签按钮来自全部画廊数据的 tags 去重集合。

请检查：

- 该标签确实在 tags 数组中
- 字段是 tags 而不是 tag
- 浏览器是否仍在使用旧缓存

## 9. 快速自检命令

```bash
cd /home/kee/web/mc-server-site
python3 -m http.server 8080 --bind 127.0.0.1
```

另开终端：

```bash
curl -I http://127.0.0.1:8080/
curl -I http://127.0.0.1:8080/css/style.css
curl -I http://127.0.0.1:8080/js/modules/init.js
curl -I http://127.0.0.1:8080/data/config.json
```

上述应返回 200（favicon.ico 例外可忽略）。
