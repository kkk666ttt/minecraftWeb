# 发布检查清单

用于每次发布前 1 分钟自检，降低线上故障概率。

## 1. 数据检查

- 所有新增记录 id 唯一
- 日期格式统一为 YYYY-MM-DD
- JSON 无语法错误（无尾逗号）
- 图片路径真实存在

## 2. 资源检查

- index.html 中引用的 JS/CSS 文件都存在
- 无历史废弃脚本引用（例如旧模块名）
- 关键资源访问返回 200

## 3. 页面检查

- 首页正常
- 插件页可筛选、可搜索、详情可弹窗
- 文档页可打开各文档
- 活动页侧边栏与详情页正常
- 画廊页可筛选、可查看、可切图
- 更新日志页最新版本显示正常

## 4. 交互检查

- 模态框可关闭
- 侧边栏可展开/折叠
- 画廊查看器缩放/拖动/切图可用

## 5. 缓存与版本

当 JS/CSS 改动后：

- 更新 index.html 对应资源版本号
- 浏览器执行硬刷新（Ctrl+F5）

## 6. 启动检查

```bash
cd /home/kee/web/mc-server-site
python3 -m http.server 8080 --bind 127.0.0.1
```

确认首页打开且控制台无红色报错。

## 7. 发布后验证

- 再次打开关键页面完整走一遍
- 检查服务端日志无大量 404
- 若有 404，确认是否仅 favicon.ico
